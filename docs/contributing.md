# Contributing to this fork

Contribution policy lives in [`../CONTRIBUTING.md`](../CONTRIBUTING.md) at the repo root. Short version:

- **Bug issues welcome** — hooks misfiring, rules contradicting themselves, broken slash commands, doc typos. Please include branch and commit hash.
- **PRs case-by-case** — bug fixes generally accepted; new rules / skills / agents need an issue first to discuss scope.
- **Forks encouraged** — adapt for your own lab. The architecture is designed for forking. Credit the upstream chain (`pedrohcgs` → `hugosantanna` → this fork → your fork) per the boilerplate in `CONTRIBUTING.md`.

For ground rules (respect the four-rule epistemic stack, no silent renames, tests for hooks, conventional commits) and the attribution chain, see the full [`CONTRIBUTING.md`](../CONTRIBUTING.md).
